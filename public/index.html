Ayuda y Guía del Visor 3D

Introducción

Esta guía explica cómo agregar nuevos vehículos (modelos 3D), imágenes de piezas y metadatos al sistema del Visor 3D. Para asegurar que todo funcione correctamente, es crucial seguir la estructura de carpetas y las convenciones de nomenclatura que se detallan a continuación.

Estructura de Carpetas y Archivos

Todo el contenido del visor se gestiona dentro de public/modulos/visor3d/. La estructura es la siguiente: 

public/modulos/visor3d/
├── data/
│   └── [ID_DEL_MODELO].json
├── imagenes/
│   └── [ID_DEL_MODELO]/
│       └── [NOMBRE_DE_LA_PIEZA].jpg
└── modelos/
    └── [ID_DEL_MODELO]/
        └── model.glb

Paso 1: Añadir un Nuevo Modelo 3D

  • Cree una nueva carpeta dentro de public/modulos/visor3d/modelos/. El nombre de esta carpeta será el ID único de su modelo (ej: nissan-versa). Use nombres cortos, en minúsculas y separados por guiones.
  • Dentro de esta nueva carpeta, coloque el archivo 3D. Debe llamarse obligatoriamente model.glb.
  • Importante: Dentro del archivo .glb, cada pieza que deba ser seleccionable o tener información asociada debe tener un nombre de malla (mesh name) único y descriptivo (ej: chasis, rueda_delantera_izquierda, motor). Estos nombres son la clave para enlazar todo lo demás.

Paso 2: Añadir Imágenes de las Piezas

  • Vaya a la carpeta public/modulos/visor3d/imagenes/.
  • Cree una nueva carpeta con el mismo ID de modelo que usó en el paso anterior (ej: nissan-versa).
  • Dentro de esta carpeta, guarde las imágenes de las piezas. El nombre de cada archivo de imagen debe coincidir exactamente con el nombre de la malla en el archivo .glb, seguido de la extensión .jpg o .png.
  • Ejemplo: Si una pieza en el .glb se llama volante, su imagen debe ser volante.jpg.

Paso 3: Crear el Archivo de Datos (Metadatos)

  • Vaya a la nueva carpeta public/modulos/visor3d/data/.
  • Cree un nuevo archivo JSON con el mismo ID de modelo (ej: nissan-versa.json).
  • Este archivo contendrá toda la información que se mostrará en las tarjetas de detalles de las piezas. La estructura es un objeto donde cada clave es el nombre de la malla del .glb.

Ejemplo de contenido para nissan-versa.json:


{
  "chasis": {
    "ID de Pieza": "CH-VSA-001",
    "Material": "Acero de alta resistencia",
    "Peso (kg)": 750,
    "Tratamiento": "Galvanizado"
  },
  "motor": {
    "ID de Pieza": "EN-VSA-HR16DE",
    "Tipo": "1.6L 4-Cilindros",
    "Potencia": "118 hp",
    "Fabricante": "Nissan"
  },
  "volante": {
    "ID de Pieza": "STR-VSA-003",
    "Material": "Cuero y Aluminio",
    "Características": "Controles multifunción"
  }
}
